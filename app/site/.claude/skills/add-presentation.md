# Skill: add-presentation

Add an embedded presentation inside a paper body. The TL;DR presentation already exists; this skill is for additional in-body presentations referenced from a `presentation` block.

## Inputs

- `paper_id` (e.g. `wp-12`)
- `presentation_id` (kebab-case, e.g. `wp-12-nexus-walkthrough`)
- `title` in EN and FR
- A list of slides. Each slide needs: `id`, `title`, `visual` type, optional `visual_config`, `caption`, `subcaption`, `text` (narration).

If any narration text would auto-generate AI slop, stop and ask the user for the script. The narration is the script ElevenLabs reads; it must be authentic.

## Steps

1. Open `data/papers/<paper_id>.en.json`.
2. Append to `embedded_presentations[]`:
   ```json
   {
     "id": "<presentation_id>",
     "owner_id": "<paper_id>",
     "title": "<EN title>",
     "locale": "en",
     "slides": [ … ]
   }
   ```
   Slide `audio_file` paths follow `public/audio/en/<presentation_id>/<NN>.mp3`. Slide `image.src` follows `public/images/<paper_id>/en/<slot>.jpg` where `slot` is `slide-<NN>` (or a meaningful name).
3. Insert a `presentation` block in the `blocks[]` array at the right point in the body:
   ```json
   { "type": "presentation", "presentation_ref": "<presentation_id>", "compact": false }
   ```
   Set `compact: true` if the presentation should render in the hero-card size.
4. Mirror everything in `data/papers/<paper_id>.fr.json`: same `id`, same slide `id`s, FR text. Audio paths go through `/fr/`.
5. If the slides reference reusable visuals (`tile-heatmap`, `stat`, `list`, etc.), no further work is needed. If they reference a bespoke visual, add `bespoke_scripts: ["components/visuals/bespoke/<paper_id>/<Name>.js"]` to the paper JSON and create the visual.
6. Run `npm run eval`.

## Visual type choice

- One stat → `stat` with `visual_config.stat_value`, `stat_label`.
- A short bulleted argument → `list` with `visual_config.items: [{ label, desc }]`.
- A direct quotation → `quote` with `visual_config.text`, `cite`.
- An image illustration → `image` with `slide.image: { src, image_prompt, style_kind: "diagram" }`. The image will be generated by `generate-images.mjs` in both locales.
- A two-column comparison → `compare` with `visual_config.left/right: { label, body }`.
- A chart → `chart` with `visual_config.kind: "mini-chart"` and chart props.
- A bespoke animation → `<paper_id>:<visual-name>` (registered through `VWVisuals.registerBespoke`).

## Validation

Load `#/paper/<paper_id>`. The embedded presentation renders inline. The play button cycles through slides. Slide numbering at top reads `01 / N`.
